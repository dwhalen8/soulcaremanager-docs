---
title: "Attaching Files"
description: "Attach documents, images, and other files to counselee records, sessions, notes, and resources."
---

You can attach files to almost anything in Soul Care Manager — a counselee's record, a session, a note, an appointment, or a resource. All uploads are private by default and subject to a 25 MB per-file size limit.

## Where you can attach files

| Attachment point | Common use |
|---|---|
| **Session** | Session notes, handouts used during the session, audio or video recordings |
| **Note** | Supporting documents for a standalone note |
| **Resource** | PDFs, worksheets, or reading material that accompany a homework resource |
| **Counselee record** | General documents related to the counselee not tied to a specific session |

## Upload a file

Every upload area in the app accepts files the same way:

- **Drag and drop** a file from your computer directly onto the upload area, or
- **Click to browse** and select a file from your file picker

Both methods work everywhere in the app.

## Supported file types

Soul Care Manager accepts any file type up to 25 MB. Common formats include:

- Documents: PDF, Word (.docx), text files
- Images: JPG, PNG, GIF
- Audio: MP3, M4A, WAV
- Video: MP4, MOV

## Previewing files

Uploaded files can be previewed inline for PDFs, images, and text files — click the file to open a preview without downloading. Other file types can be downloaded and opened in the appropriate application.

## File privacy

All uploaded files are **private by default**. A file attached to a session is visible only to the counselor who uploaded it. Counselees cannot see attachments unless you explicitly share the file with them.

To share a file with a counselee, see [Sharing Files](/files/sharing-files).

## Deleting a file

To remove an uploaded file, open the file from wherever it's attached and use the delete option. Deleted files are removed from the system permanently — there is no recovery.

<Warning>
  File deletion is permanent. Make sure you no longer need the file before deleting it.
</Warning>
